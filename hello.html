Most Important Git Command
  1. Here, I will post some common git commands and their explanations.
  2. First is: git clone "url" . After moving to a location of your choosing, use this command to download the entire repository into the folder.
  3. Second is: git add "file name". I can save a file locally, but I also need to save or "commit" the file to git, so I can use git's version control software. This command pushes the file to a staging area. Mutliple files can be pushed to the staging area.
  4. Third is: git commit. This command takes all the files on the staging area and saves them. All of this is done locally on the computer. However, by commiting, you let git know this is a version that you might need later.
  5. Fourth is: git push. This command takes all the commits you have done locally and add those commits to the online repository. I think it has to be the case that the folder with all your files has to be a future of a cloned file.
  6. Fifth is: git branch. This command just lets you know the different branches in your repository and the branch you are currently on. Master is the main branch.
  7. Sixth is: git branch "name". This creates a new branch. git checkout "name": changes the branch to the new branch. git merge "name": merges the master and a single branch.
  8. Seventh is: git log. This shows all the changes/commits you have made.
  9. git fetch + git merge origin/master = git pull: this updates the local folder to current repository online and updates the Head to the current Head online.
